Part 1: Poor Man's AR

Overview

This project involved exploring naive augmented reality. By employing video tracking, manipulating 2D image and 3D world coordinates, and utilizing camera projection matrices, I was able to superimpose a 3D cube onto a video. This process effectively simulated a virtual reality environment with a straightforward method.

Tracking

I started with taking a video of a small white box, onto which I mapped 3cm x 3cm squares to form a grid. Using photoshop, I then manually selected 22 keypoints (vertices of the grid) from the first frame of the video and used an off the shelf tracker to track these points throughout the entirety of the video. I decided to use OpenCV's Median Flow Tracker and created a separate tracker for each point. I used an 8 x 8 patch size around each point as the bounding box. Calibrating the Camera

I started off by defining the bottom corner of the cuboid as (0, 0, 0) in world coordinates and converted all the selected 2D points in the first frame into 3D world coordinates using physical measurements of the cube. Once I had both the 3D and 2D points for each frame, I computed a projection matrix for the camera to convert the 3D world coordinates into the 2D image coordinates for each frame. I worked with homogeneous coordinates and used the 2 equations below to solve for the projection matrix using the least squares method:

Projecting a Cube in the Scene

I accomplished the projection of a cube into a scene by first establishing the object coordinates for the cube. Utilizing the calibration matrix specific to each frame, I projected these defined points accordingly and used a drawing function to draw a cube on these projected axes. This process was meticulously repeated across each individual frame, thereby enabling the continuous propagation of the cube's presence throughout the entire sequence of frames. This method ensured that the cube was consistently and accurately integrated into the scene in each frame, creating a seamless and dynamic visual effect.

Part 2: A Neural Algorithm of Artistic Style

Overview

This project focuses on re-implementing the model presented in A Neural Algorithm of Artistic Style and involves transferring the style features of one particular image onto another subject image, the content image via neural network learning of style and content weights.

Implementation Details

The paper discusses adapting an open-source 19-layer VGG network, originally trained on the ImageNet dataset. Key modifications and enhancements were made to the original network design:

  • Network Configuration: The adapted network utilizes only the first 16 convolutional layers and 5 pooling layers from the original architecture.
  • Pooling Technique Change: To enhance gradient flow and achieve more visually appealing results, max-pooling operations were replaced with average pooling.
  • Layer Selection for Style and Content Reconstruction: The paper presents five different configurations (labeled a to e) for selecting subsets of CNN layers for style reconstruction. In this reimplementation, option (e) was chosen, which involves using layers conv1_1, conv2_1, conv3_1, conv4_1, and conv5_1 for style reconstruction. For content reconstruction, the conv5_1 layer is utilized.
  • Final Network Structure: The final network configuration includes 16 CNN layers, each followed by a ReLU layer and an average pooling layer, except the last one. Notably, there are no fully connected layers following the final CNN layer.

Two kinds of losses are used for the style transfer process: style loss and content loss. The style loss is computer over all 5 selected conv layers. It involves computing the mean squared error between the Gram matrix of the style image and that of the input image. Each of the 5 layers' loss is weighted equally. The content loss is only calculated on the conv5_1 layer and involves calculating the mean squared error between the input image and the original content image. More details on the losses can be found in the paper's method section.

The vgg19 network was modified so that output features from each of the 5 conv layers are returned in the forward pass. The style and content loss is then calculated for each of these 5 output features simultaneously and the 2 losses are then weighted. The paper defines this ratio of content loss weight to style loss weight as the 'alpha/ beta' ratio. I managed to get the best results by using a ratio of 1e-6 for the final loss (content_loss_weight = 1, style_loss_weight=1000000). The Adam optimizer is used to decrease the loss and the input image gets transformed as a process of this optimization. I used a learning rate of 0.1 for 2000 iterations to achieve my results.
